Token Coach verdict language

Loop 2 deliverable — 3 variants per string, sample numbers filled in. Winners (orange) are locked and shipping now.

How to swap a winner: edit LOCKED in desktop/lib/token-coach/templates.js — e.g. r1: { what: 2 } ships variant #2. Rules and UI read through render(); nothing else changes. Tests: cd desktop && npm test.

R1 — Long-thread bleed

one card per offending chat

Title (max 8 words)

0

One long chat drained your limit

Locked
1

Marathon chat ate your week

2

Old messages billed you again and again

What happened

0

One Claude Code chat ran 50 messages — every new message re-sent the whole conversation again, up to 260K tokens each time.

1

You kept one Claude Code chat alive for 50 messages. Each reply paid to re-read everything above it — up to 260K tokens.

Locked
2

One marathon Claude Code chat hit 50 messages; by the end every question dragged 260K tokens of old chat along with it.

Fix

0

Start a fresh chat when you switch tasks.

Locked
1

Open a new chat for each new task — old history costs you every message.

2

When the task changes, change the chat.

Cards only fire on uncached re-sending — a long thread with healthy caching stays silent, so the fix never contradicts R3.

R3 — Cache misses

one card per agent (habit)

Title

0

Your chats kept paying full price

Locked
1

Repeat content billed at full price

2

The discount never kicked in

What happened

0

4 Claude Code sessions kept paying full price — only 5% of what the model re-read came from its memory, the rest billed fresh: 2.5M tokens.

Locked
1

Claude Code re-read your project from scratch in 4 sessions instead of using its short-term memory — 2.5M tokens paid twice.

2

In 4 Claude Code sessions the discount for repeated content barely kicked in (5%) — 2.5M tokens billed at full price.

Fix

0

Keep one chat going per task instead of stopping and restarting.

1

Work in longer stretches — pauses over 5 minutes make the model forget and re-bill.

2

Don't restart chats mid-task; each restart re-bills your whole project.

Locked
Fix #2 chosen over #1: the 5-minute detail is true but reads as trivia; #2 names the billing consequence and never contradicts R1's "fresh chat per task".

Locked

Fix #1 chosen: it points at maxxToken's own menu bar — the product teaches the user to use the product.
